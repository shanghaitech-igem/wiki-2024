# Introduction

This tutorial is meant to help you get started with writing your own MDX page for the our iGEM wiki.

[MDX](https://mdxjs.com/) is just a superset of [Markdown](https://www.markdownguide.org/), which means beyond the basic markdown syntax, you can also include "JSX Component" and "frontmatter" in your MDX file.

# MD Support

## Links

### Creating Links

- Syntax:

  ```markdown
  [In Page Jump Test](#section-id)

  [In Site Jump Test](/path)
  ```

- Example:

  - Code:

    ```markdown
    [Go to GFM Section](#github-flavored-markdown-gfm)

    [Go to Description Page](/description)
    ```

  - Result:

    [Go to GFM Section](#github-flavored-markdown-gfm)

    [Go to Description Page](/description)

- Notes:
  _You can hover your cursor over the headings to see the its URL, where its `section-id` is in it._

# MDX Support

## Frontmatter

### Overview

Frontmatter is pretty much just a configuration block for the page, at the beginning of your MDX file.

Like this:

```yaml
---
title: "Title"
favicon: "path/to/favicon"
slug: "path/to/page"
---
```

Currently, the following frontmatter fields are supported:

- `title`: The big title on the top of the page.

- `slug`: The URL path of the page.

- `favicon` (optional) : The favicon of the page, which will be shown in the browser tab.

- `tracer` (optional) : The cursor tracer of the page, which will following user's cursor.

### An Example of Frontmatter

The frontmatter of the current page is:

```yaml
---
title: "Writing Tutorial"
favicon: "server/test/igem-favicon.svg"
slug: "docs/writing"
---
```

## JSX Components

### Overview

Currently, following components are supported in our Wiki MDX pages:

- `Image`: Showing images and GIF with optional caption.

- `Plot`: Use Plotly to create interactive plots.

- `Modal`: Create a modal with a button, once the button is clicked, the modal will pop up.

- `PDF`: Embed PDF files with native browser reader.

- `Video`: Embed videos with a player.

- `Audio`: Embed audio files with a player.

- `Model`: Embed 3D models with a interactive viewer.

- `Collapse`: Create a collapsible container.

### An Example of JSX Component

JSX component are components that could take props.

- **Syntax:**
  ```jsx
  <ComponentName prop1="value1" prop2="value2" />
  ```

### Image Component

Actually, simply using [standard markdown image syntax](http://localhost:8000/docs/writing/#images) is already calling the `Image` component.

If you want to specify the height and width manually or simply want to directly use the `Image` component (**which is not recommended**), here is the syntax:

- **Syntax:**
  ```jsx
  <Image
    src="image-url"
    alt="Alt Text (optional)"
    title="Caption Text (optional)"
    height="Height (optional)"
    width="Width (optional)"
  />
  ```

- **Example:**
  - Code:

    ```jsx
    <Image
      src="server/test/hula-loop-octodex-gif.webp"
      alt="Minion"
      title="Minion"
      height="300"
      width="300"
    />
    ```
  - Result:

    <Image
      src="server/test/hula-loop-octodex-gif.webp"
      alt="Minion"
      title="Minion"
      height="300"
      width="300"
    />

### Plot Component

Plot component use `plotly.js` to create interactive plots, **the data need to be store externally and imported**.

- **Syntax:**

  ```jsx
  import data from "path/to/data";

  <Plot {...data} />;
  ```

import data_1 from "/test/data/testData-1";

- **Example:**
  - Code:
    ```jsx
    import data_1 from "/test/data/testData-1";

    <Plot {...data_1} />
    ```

  - Result:

    <Plot {...data_1} />

More details about this component are [here](/docs/plot) and **need to be read if you want to actually use it**.

### Modal Component

Modal is a component that could create a modal with a button, once the button is clicked, the modal will pop up.

- **Props:**

  - `title` (optional): A string to customize the modal title. Defaults to "Modal" if not specified.

  - `triggerText` (optional): A string to customize the button text. Defaults to "Open Modal" if not specified.

- **Syntax:**

  ```jsx
  <Modal title="Markdown Test" triggerText="Click Me">

    # Modal Content

    Content inside is a new container that could contain any markdown syntax.

  </Modal>
  ```

- **Example:**

  - **Code:**

    ```jsx
    <Modal title="Markdown Test" triggerText="Click Me">

      # Modal Content

      Content inside is a new container that could contain any markdown syntax.

    </Modal>
    ```

  - **Result:**

    <Modal title="Modal Test" triggerText="Click Me">

      # Modal Content

      Content inside is a new container that could contain any markdown syntax.

    </Modal>

- Notes:
  _Please keep to indent the contents in the `<Modal>`, the indentation is essential_

### Video Component

Important: Videos could not be uploaded by `igem-cdn`, you should go to [iGEM Video Universe](video.igem.org), and **it requires full embed URL of the video**.

- **Syntax:**

  ```jsx
  <PDF src="full-path/to/embed-video" />
  ```
  Your URL should look like this:
  ```url
  https://video.igem.org/videos/embed/<some-random-number>
  ```
  
- **Example:**

  - **Code:**
    ```jsx
    <Video src="https://video.igem.org/videos/embed/426c83d7-d298-400e-b534-bddecd176aab" />
    ```
  - **Result:**
    <Video src="https://video.igem.org/videos/embed/426c83d7-d298-400e-b534-bddecd176aab" />

### Audios Component

  Your URL should look like this:

- **Example:**

### Model Component

Model component renders 3D models in real time, **the data need to be store externally on the iGEM CDN Server**.

- **Syntax:**

  ```jsx
  <Model src="path/to/model" />
  ```

- **Example:**

  - **Code:**
    ```jsx
    <Model title="A Protein" src="server/test/test-protein.stl" />
    ```
  
  - **Result:**
    <Model title="A Protein" src="server/test/test-protein.stl" />


### Collapse Component

The Collapse component creates a collapsible container, which can be used to hide any MDX content.

- **Props:**

  - `title` (optional): A string to customize the button text. If provided, the button will display "Expand: title" or "Collapse: title". Defaults to "Expand" and "Collapse" if not specified.

- **Syntax:**

  ```jsx
  <Collapse title="Your Title">
    # Content

    Content inside is a new container that could contain any markdown syntax.
  </Collapse>
  ```

- **Example:**

  - **Code:**

    ```jsx
    <Collapse title="Section">
      # Content

      Content inside is a new container that could contain any markdown syntax.

      <Model title="A Protein" src="server/test/test-protein.stl" />
    </Collapse>
    ```

  - **Result:**

    <Collapse title="Section">
      # Content

      Content inside is a new container that could contain any markdown syntax.

      <Model title="A Protein" src="server/test/test-protein.stl" />
    </Collapse>

- Notes:
  _Please keep to indent the contents in the `<Modal>`, the indentation is essential_
